# Case 1 問題構造の全体像

## 概要

**ファイル**: `case1.gms`（314KB, 7,082行）
**問題種別**: NLP（非線形計画問題） — 整数変数なし
**目的**: 利益（= 製品売上 − 原油購入コスト）の最大化
**ソルバー指示**: `Solve m using MINLP maximizing x3573;`

### 目的関数（e3428）の構造

```
e3428: -Σ(製品価格 × 製品流量) + Σ(原料コスト × 原料流量) + x3573 = 0
```

整理すると `x3573 = Σ(製品価格 × 製品流量) − Σ(原料コスト × 原料流量)` であり、`maximize x3573` = 利益の最大化となる。

## 制約条件の構造（e1 〜 e3428）

### 制約の分類

| 種類 | 本数 | 記号 | 内容 |
|------|------|------|------|
| 双線形等式 | 289本 | =E= | 品質追跡（流量×品質=品質質量流量） |

| 上限不等式 | 908本 | =L= | 品質上限、容量上限、流量上限 |

| **合計** | **3,428本** | | |

### 制約の詳細

#### 1. 双線形等式（e1〜e289付近）— 品質追跡

```
e1:  x3481 * x1293 - x1095 = 0
```

意味: `品質比率(x3481) × 品質値(x1293) = 品質質量流量(x1095)`

ミキサーでストリームA（流量fA、硫黄濃度qA）とストリームB（流量fB、硫黄濃度qB）を混合したとき、出口の硫黄濃度は `(fA×qA + fB×qB) / (fA+fB)` となる。この分子の `fA×qA` が双線形項であり、**問題を非凸にしている原因**。

#### 2. 線形等式 — マスバランス（e534〜付近）

```
e534:  x1 - x366 - x367 = 0      （入力 = 出力の和）
e538:  x239 - x604 - ... - x609 = 0  （スプリッターの分配）
```

意味: 各装置ノードで「入る流量の合計 = 出る流量の合計」。物質保存則。

#### 3. 線形等式 — 収率関係（e290〜付近）

```
e290:  x12 - x1211 = 0
```

処理装置の入力流量と出力流量の関係。固定収率装置（Upf）では、出力 = 収率係数 × 入力。

#### 4. 不等式 — 流量・品質の上下限（e3380〜e3428付近）

```
e3380:  x14 >= 2         （ストリームの最低流量）
e3386:  x3  <= 0.061     （ストリームの最大流量）
e3388:  x11 <= 0.8       （製品の最大販売量）
```

## Pythonへの移植に関する注記

`case1.gms`はスカラー形式のため、2つの移植方法がある。

**方法A（スカラー形式の直接パース）**: `case1.gms`の3,573個の変数宣言、3,428本の制約式、変数上下限をそのまま正規表現でパースし、Pyomoの変数・制約に変換する。意味は分からないが、数値的に同一の問題を再現できる。

**方法B（意味ベースの再構築）**: `all_sets.txt`と`all_parameters.txt`をパースし、論文の数式に基づいてPyomoで添字付きモデルを構築する。マスバランス、品質追跡、収率、容量制約を個別に実装する。理解は深まるが、論文の定式化を正確に読み解く必要がある。

いずれの方法でも、ソルバーは`IPOPT`（局所最適、無料）→ `SCIP`（大域的、オープンソース）の順で試すのが推奨。
